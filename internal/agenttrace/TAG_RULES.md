# Tag Rules

Trace tags are generated by Go code, not by runtime configuration. Source-level
custom tag rules live in `internal/agenttrace/tagrules.go`.

Use tag rules for fixed, reusable turn-level filters. Keep them deterministic,
low-cardinality, and secret-free. Do not turn a matched string, prompt text,
command output, file path, URL, session ID, trace ID, or exact MCP tool name into
a tag.

## Add a request text tag

For a fixed tag when the user's request contains one of a few strings, add a rule
to `tagRules`:

```go
var tagRules = []TagRule{
	fixedRequestContainsRule(
		"request_mentions_mcp",
		"request:mentions_mcp",
		"mcp",
		"model context protocol",
	),
}
```

This emits `request:mentions_mcp` when the user request contains `mcp` or
`model context protocol`. The matched string itself is not exported.

## Add a custom rule

For logic that needs the parsed turn or existing rollup counts, add a small
function and include it in `tagRules`:

```go
var tagRules = []TagRule{
	requestUsesNetworkCommandRule,
}

var requestUsesNetworkCommandRule = TagRule{
	Name: "request_uses_network_command",
	Tags: func(turn Turn, rollup InsightRollup) []string {
		if rollup.CommandKindCounts[CommandKindNetwork] > 0 {
			return []string{"command:network_used"}
		}
		return nil
	},
}
```

Rules may return several tags, but each tag must be a fixed value with only
lowercase letters, digits, `:`, `_`, `-`, or `.`. Invalid tags are ignored.

After changing tag rules, update tests or golden fixtures when the exported tag
contract changes, then run:

```sh
go test ./... -count=1
git diff --check
```
